# Порівняння рендерів до та після оптимізації

## Система відстеження рендерів

Додано систему відстеження рендерів через `utils/render-tracker.ts`, яка:
- Відстежує кількість рендерів для кожного компонента
- Логує статистику в консоль
- Показує детальну інформацію про ререндери

## Оптимізації, які були застосовані

### 1. React.memo для PetCard
- Компонент ререндериться тільки при зміні пропсів
- Кастомна функція порівняння для точного контролю

### 2. useCallback для функцій
- `handlePetPress` - стабільна функція
- `handleFavoritePress` - стабільна функція
- `renderPetItem` - стабільна функція
- Всі інші render функції - стабільні

### 3. useMemo для обчислень
- `typeLabel` - обчислюється тільки при зміні `type`
- `dynamicStyles` - мемоізовані стилі
- `cardStyles` - мемоізовані стилі

## Як перевірити рендери

1. Відкрийте консоль розробника
2. Завантажте додаток
3. Перегляньте логи:
   - `[RenderTracker]` - показує кількість рендерів
   - `[PetCard]` - показує деталі рендерів карток
   - `[HomeScreen]` - показує рендери головного екрану
4. Через 2 секунди після завантаження автоматично виведеться підсумкова статистика

## Очікувані результати

### До оптимізації:
- PetCard рендериться при кожній зміні стану (навіть якщо пропси не змінилися)
- Всі картки ререндеряться при зміні однієї улюбленої
- Функції створюються заново при кожному рендері

### Після оптимізації:
- PetCard рендериться тільки при зміні своїх пропсів
- Тільки картка зі зміненим isFavorite ререндериться
- Функції стабільні і не викликають ререндери
